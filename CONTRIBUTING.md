# Contributing to VizKit

Thanks for taking the time to help improve VizKit.

## Ways to contribute
- Report bugs
- Suggest new built-in tools
- Improve menu organization
- Refine documentation
- Add polish to visuals and packaging

## Workflow
1. Fork the repository.
2. Create a feature branch.
3. Make focused changes.
4. Test the batch file on Windows 10/11.
5. Open a Pull Request.

## Guidelines
- Keep the toolkit Windows-native.
- Avoid adding external dependencies unless absolutely necessary.
- Prefer readable batch code and clear labels.
- Update the README when features change.

## Notes
If you are proposing a new command, make sure it is safe, useful, and consistent with the portable-toolkit design.
